PROJECT 1

AI 문서관리시스템 구축

수행기간 2025.06 ~ 2025.10 · 역할: 문서관리 프로세스 기획 / LLM 프롬프트 설계

문제 정의

발전소 수 증가로 문서가 급증했으나 구글 드라이브 내 분류 체계 없이 저장되어 최신 문서 식별이 어렵고, 중복 파일 증가 및 공정 일정 지연 발생.


가설: AI가 문서를 자동 분류·태깅하면, 담당자의 수동 관리 없이도 최신 문서를 즉시 파악할 수 있다.

실행
  • 발전소 공정 단계별 문서 유형 분류 기준 설계
  • OCR 기반 텍스트 추출 구조 기획
  • LLM 프롬프트 규칙 설계
  • Google Drive + 내부 어드민 [업로드→검증→승인→DB 입력] 워크플로우 기획
성과
📉 문서 검수 시간 70% 절감
📋 발전소별 문서 관리 체계 표준화
🏆 사내 최초 AI 기반 서비스 기획 사례 → AI 솔루션 내재화 레퍼런스 수립
PROJECT 2

발전소 운영 어드민 서비스 기획

수행기간 2023.09 ~ 2025.10 · 역할: 유저 인터뷰 / IA 설계 / Wireframe / MVP 사이클 주도

문제 정의

기존 어드민은 발전소·조합 관리 중심으로만 구성되어 영업·리스크·정산 업무는 수기 처리. 사용자 관점이 전혀 고려되지 않은 UI로 비효율이 누적.


Case 1. 전자계약 도입

계약서 수령 기간 3일 → 3시간 · 전자계약 비중 90% 확대


Case 2. 리스크 검토 자동화

NICE·쿠콘 API 연동 → 리스크 검토 시간 10건 기준 1시간 → 5분 단축

실행
  • 영업·구축·운영·정산 4개 부서 인터뷰 → Pain Point 도출
  • IA 재설계: 2개 영역 → 5개 영역으로 확장
  • 이폼싸인 API 연동 기획
  • NICE 신용평가 API·등기부등본 API 연동 스펙 정의
  • 우선순위 기반 MVP 도입 → 피드백 → 개선 사이클 주도
성과
⚡ 리스크 검토 1시간 → 5분
📝 전자계약 비중 90% 확대
⏱ 계약서 수령 3일 → 3시간

PROJECT 3

RPA 구축 및 내재화

수행기간 2024.03 ~ 2025.10 · 역할: 자동화 우선순위 선정 / 프로세스 설계·검증 / UAT 조율

문제 정의

발전소 수 증가에 따라 각 부서의 수기 반복업무가 비례적으로 늘어남. 인력 충원보다 자동화를 통한 비용 절감이 필요한 상황.


우선순위 기준:
1순위: 매출에 직접 영향을 주는 업무
2순위: 지자체 등 외부 일정에 영향받는 업무
3순위: 내부 이해관계자가 많이 엮인 업무

실행
  • 현업 인터뷰 → 자동화 가능 업무 파악
  • 총 34개 태스크 중 21개 담당, 프로세스 설계·검증
  • 개발팀과 UAT 조율 → 현업자 피드백 반영 후 재검증
  • 한국전력 청구 이메일 1,400여건 파싱 자동화
  • REC 발급 및 세금계산서 발행 자동화 (발전소 1,300여개)
성과
⏳ 연간 1,200시간 절감
📈 발전소 운영 자동화율 75%
📧 청구서 파싱: 2일 → 4시간
📋 REC 발행: 5일 → 8시간
SIDE PROJECT

bookswap — 교환독서 모임 관리 플랫폼

수행기간 2026.05 · 역할: 서비스 기획 / UI 디자인 / 프론트엔드 개발 / 배포 (1인 풀사이클) · 서비스 바로가기 →

문제 정의

직접 교환독서 모임을 운영하면서 느낀 반복적인 불편함에서 출발.


• 카카오톡으로 교환 현황 구두 공유

• 반납일을 기억에만 의존

• 책 제목 직접 타이핑

• 교환 이력 기록 없음

• 책 감상 나눌 공간 없음


가설: 교환 순환과 이력을 자동화하면, 모임 운영 부담 없이 독서에만 집중할 수 있다.

실행
  • 직접 겪은 Pain Point 기반 기능 우선순위 정의
  • IA 설계: 홈 / 모임 / 내 기록 / 통계 / 마이페이지
  • Vanilla JS + Supabase(PostgreSQL) 단일 파일 SPA 구현
  • 네이버 책 검색 API 연동 (Vercel Serverless 프록시)
  • 교환 주기 기반 D-day 자동 계산 + 배너 알림
  • 멤버 배열 circular shift로 순환 자동화
  • 생성자 전용 모임 설정 (멤버 관리 / 정보 수정 / 모임 종료)
  • 고유 코드 기반 타기기 복원 기능
  • PWA 설정으로 모바일 앱처럼 설치 가능
성과
📚 실사용 모임 4개 운영 중 · 멤버 17명
⚡ 기획 → 배포 2주 이내 완성
